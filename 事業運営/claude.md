# Luna Pos — 事業運営プロジェクト

## このプロジェクトについて

Luna Pos はキャバクラ・ナイトクラブ向け POS SaaS。本プロジェクトは**開発以外**の事業運営タスクを管理する。
開発（POS / HP / LP / 管理画面）は同階層の各ディレクトリで進行中。

- 事業形態: 個人事業主（開業届提出済み）
- βリリース目標: 2026年3月末（新宿アップスでテスト稼働）
- 正式リリース目標: 2026年4月末
- 月額料金: ¥30,000（税込）ワンプラン

## プロジェクト構成

本ファイルは `事業運営/` に配置。同階層の兄弟プロジェクトを含めた luna/ 全体の構成は以下の通り。

```
luna/
├── lunapos-floor/         ← POS フロア画面（Vite + Swift WebView）※メイン開発
├── lunapos-cast/          ← キャスト向け画面（Vite）
├── lunapos-admin/         ← オーナー管理画面（Vite）
├── hp/                    ← ホームページ（Next.js）
├── lp/                    ← ランディングページ（Vite）
├── partner/               ← パートナープログラム関連資料
├── trust/                 ← 競合 TRUST の資料（パンフレット等）
├── 規約/                  ← 法務文書（利用規約・PP・契約書等 .docx）
├── images/                ← ロゴ・素材画像
├── メディア文作成/         ← メディア向け文章
└── 事業運営/              ← 本プロジェクト（事業運営タスク管理）
    └── claude.md          ← 本ファイル
```

## 基本ルール

- 言語: 日本語で作業・出力する（コード内コメントも日本語可）
- 文書作成時は Markdown で作成し、必要に応じて後から docx/pdf に変換する
- 法務文書は「弁護士レビュー前のドラフト」として作成する。最終版ではない旨を明記すること
- 金額は全て税込で記載する
- 日付は YYYY-MM-DD 形式を使用する

## claude.md の自動更新ルール

**作業のたびに本ファイルを最新状態に保つこと。** 具体的には以下のタイミングで更新する:

- タスクを完了したら → 該当タスクを `完了済みタスク` に移動し、`タスク一覧` から削除する
- 新しいタスクが発生したら → 適切な優先度セクションに追加する
- 方針・料金・KPI 等に変更があったら → 該当セクションを更新する
- ディレクトリ構成が変わったら → `プロジェクト構成` を更新する
- 期限が変更・追加されたら → `重要な期限` テーブルを更新する
- 新しい競合情報やリスクが判明したら → 該当セクションに追記する
- 予算に変更があったら → `予算` セクションを更新する

## リスクと対策

| リスク | 対策 |
|---|---|
| App Store審査リジェクト | ガイドライン事前確認。代替: Business Manager / Web Clip |
| 競合の価格引き下げ | 品質・UI・サポートの価値で勝負。低価格競争に乗らない |
| β版での重大バグ | βテスト期間を活用。障害対応フロー事前策定 |
| 初期顧客が獲得できない | β版無料で実績づくり。紹介経由の営業を最優先 |
| 資金ショート | 補助金・融資の複線化。固定費を最小限に |
| 個人情報漏洩 | RLS徹底。暗号化。定期セキュリティレビュー |
| 法的文書の不備 | 弁護士レビュー予算化。最低でも利用規約は専門家確認 |

## 注意事項

1. 法的文書は専門家レビュー前のドラフトとして扱う
2. App Store審査はナイトクラブ関連で厳しくなる可能性あり（ガイドライン 1.1.4 / 5.2.1）
3. 持続化補助金の締切を厳守（GビズID取得に数週間かかる）
4. β導入は無償でも契約書を必ず交わす
5. SNSは最低週2回投稿を習慣化（放置は逆効果）
6. キャスト・顧客データは機微情報 — RLS・暗号化・アクセスログを徹底
7. 金額記載は全て税込統一
