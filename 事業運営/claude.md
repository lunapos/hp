# Luna Pos — 事業運営プロジェクト

## このプロジェクトについて

Luna Pos はキャバクラ・ナイトクラブ向け POS SaaS。本プロジェクトは**開発以外**の事業運営タスクを管理する。
開発（POS / HP / LP / 管理画面）は同階層の各ディレクトリで進行中。

- 事業形態: 個人事業主（開業届提出済み）
- βリリース目標: 2026年3月末（新宿アップスでテスト稼働）
- 正式リリース目標: 2026年4月末
- 月額料金: ¥30,000（税込）ワンプラン

## プロジェクト構成

本ファイルは `事業運営/` に配置。同階層の兄弟プロジェクトを含めた luna/ 全体の構成は以下の通り。

```
luna/
├── lunapos-floor/         ← POS フロア画面（Vite + Swift WebView）※メイン開発
├── lunapos-cast/          ← キャスト向け画面（Vite）
├── lunapos-admin/         ← オーナー管理画面（Vite）
├── hp/                    ← ホームページ（Next.js）
├── lp/                    ← ランディングページ（Vite）
├── partner/               ← パートナープログラム関連資料
├── trust/                 ← 競合 TRUST の資料（パンフレット等）
├── 規約/                  ← 法務文書（利用規約・PP・契約書等 .docx）
├── images/                ← ロゴ・素材画像
├── marketing/article/     ← オウンドメディア記事
├── SNS運用/               ← SNS アカウント運用（X / Instagram / TikTok）
└── 事業運営/              ← 本プロジェクト（事業運営タスク管理）

```

## 基本ルール

- 言語: 日本語で作業・出力する（コード内コメントも日本語可）
- 文書作成時は Markdown で作成し、必要に応じて後から docx/pdf に変換する
- 法務文書は「弁護士レビュー前のドラフト」として作成する。最終版ではない旨を明記すること
- 金額は全て税込で記載する
- 日付は YYYY-MM-DD 形式を使用する

## claude.md の自動更新ルール

**作業のたびに本ファイルを最新状態に保つこと。** 具体的には以下のタイミングで更新する:

### Dシステムについて（要注意競合）

- **5,000店舗導入**はTRUSTの10倍。実質的な業界最大手の可能性
- スモールビジネスプラン（月¥15,000 税込）でスナック・ガールズバー向けの低価格帯を押さえている
- ハンディ機能が追加料金なし
- スマホでリアルタイム売上確認可能
- → Luna Posの¥30,000はDシステムの2倍。キャバクラ向けフル機能での差別化が必要

### 差別化メッセージ（LP・営業資料・事業計画で一貫して使用）

1. **モダンUI/UX** — SwiftUI ネイティブで滑らかな操作感
2. **透明な料金体系** — Webで料金公開、「要問合せ」不要
3. **Web管理画面** — 外出先からリアルタイムに売上・給与確認
4. **不正防止が標準搭載** — TRUSTは月¥60,258のプレミアムプランのみ。Luna Posは全ユーザーに提供
5. **インボイス対応** — 適格請求書の要件を満たすレシート・領収書出力

### フェーズ別ターゲット戦略

1. **Phase 1（ローンチ〜6ヶ月）**: キャバクラに集中。機能が最も刺さるセグメント
2. **Phase 2（6ヶ月〜）**: **スナック市場への展開**。最大セグメント（5〜6万店舗）。料金体系を簡素化した「スナックモード」が必要。時間制・ボトル制への対応。改正風営法（2025年6月施行）で正規化の流れが追い風
3. **Phase 3（1年〜）**: ガールズバー・クラブ・ラウンジへ拡大

### スナック市場の法的背景

スナックの営業形態は3パターン:

1. **飲食店営業許可のみ**（0時まで・接待なし）— 最もシンプル
2. **深夜酒類提供飲食店営業届出**（0時以降・接待なし）— 多くのスナックがこれ
3. **風俗営業1号許可**（接待あり）— キャバクラと同じ

**実態**: ママが隣に座ってお酌・おしゃべり・デュエット = 風営法上の「接待」だが、多くのスナックは②の届出のみで営業（グレー〜違反）。最近は摘発が増加傾向。

**2025年6月 改正風営法が施行済み**:
- 無許可営業の罰則が大幅強化: 個人1,000万円以下 / 法人3億円以下の罰金
- 警察が本腰を入れて摘発する方向
- 「色恋営業」「料金の虚偽説明」「注文していないドリンクの強制提供」が明確に規制対象
- → グレー営業のスナックは「正規化するか、閉店するか」の選択を迫られている

**Luna Posへの示唆**:
- 改正風営法の施行でスナックの正規化ニーズが急増 → POS導入の追い風
- スナック向け展開時は、風営法コンプライアンス支援とセットで訴求すると効果的（「正しく営業する店舗を支援する」ポジション）
- Dシステムが月¥15,000のスモールビジネスプランでこの市場を押さえている点は要研究
- フリーミアム（500会計無料）は小規模スナック（月商150万円・10席）にとって非常に魅力的

**スナックの売上規模（10席の場合）**:
- 月商: 約130〜170万円 / 年商: 約1,600〜2,000万円
- ママの年収: 300〜600万円
- POS月額¥30,000は月商の約2%。¥15,000なら1%。フリーミアムなら無料で使い続けられる可能性大

## ボトムアップ導入戦略（PLG）

**核心: 「店舗導入」の前に「個人利用」のステップを作る。**

ナイト業界では POS の導入決裁権はオーナーにあるが、日常的に POS を触るのは黒服（フロアスタッフ）やキャスト。決定権のない現場の人間がボトムアップで導入を推進できる環境を整える。SaaS 業界の PLG（Product-Led Growth）をナイト業界に適用する。

### 店舗の意思決定構造

| 役割 | 決定権 | Luna Pos との接点 |
|---|---|---|
| オーナー | 最終決裁 | 売上・経営データ |
| 店長/マネージャー | 運用判断・オーナーへの提案 | 日常のオペレーション全体 |
| 黒服（フロアスタッフ） | なし | POS 操作の主体 |
| キャスト | なし | 自分の売上・指名の確認 |

### 施策

1. **キャストアプリの「個人モード」**（最重要）
   - lunapos-cast を店舗導入なしで使えるようにする
   - 自分の売上・指名数・バック額を手動で記録できる「個人メモ」モード
   - オーナー許可不要・無料で使い始められる → 導入ハードルゼロ
   - キャストが個人で使い始める → 「店全体で入れたらもっと便利」→ オーナーに話が上がる
   - → 詳細は `lunapos-cast/CLAUDE.md` を参照

2. **「オーナーに見せるデモ」の整備**
   - 30秒デモ動画（スマホで見せられる長さ）
   - 「オーナーさんに見せてください」用の1枚ペラ（QRコード付き）
   - LINE で転送できる LP（オーナーが LINE で情報を受け取る文化に合わせる）

3. **フリーミアム（月500会計無料）を「試しに入れてみて」の武器にする**
   - 現場スタッフが言いやすいセリフ: 「無料で使える POS あるんですけど、試しに入れてみませんか？合わなかったらやめればいいだけなんで」
   - 初期費用 ¥0 + 月額 ¥0（500会計まで）= オーナーのリスクがゼロ
   - 500会計で足りなくなったら、もう業務に組み込まれている状態

4. **紹介インセンティブを「提案した本人」に渡す**
   - キャストや黒服が自店舗のオーナーに紹介 → 導入されたら紹介者本人に報酬
   - 例: 導入後3ヶ月継続で ¥10,000 キャッシュバック（紹介者へ）
   - オーナーではなく現場の人が動く動機を作る

5. **SNS でキャスト・黒服コミュニティに直接リーチ**
   - キャストは個人で SNS（Instagram / TikTok / X）を活発にやっている
   - 「売上管理アプリ」として個人向けに訴求 → 店舗導入への導線
   - → 詳細は `SNS運用/CLAUDE.md` を参照

### 施策の優先度

| 施策 | 工数 | 効果 | 優先度 |
|---|---|---|---|
| キャストアプリの個人モード | 中 | 高（入口になる） | 最優先 |
| 30秒デモ動画 | 小 | 中 | 高 |
| オーナー向け1枚ペラ | 小 | 中 | 高 |
| 紹介インセンティブ設計 | 小 | 高 | 高 |
| SNS キャスト向けコンテンツ | 小 | 中〜高 | 中 |

2. App Store審査はナイトクラブ関連で厳しくなる可能性あり（ガイドライン 1.1.4 / 5.2.1）
