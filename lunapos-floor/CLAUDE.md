# POS（営業中操作アプリ）プロジェクト指示書

---

## 1. アプリ概要

**アプリ名**: Luna Pos Floor（営業中操作）

**目的**: 営業時間中にボーイ（フロアスタッフ）が使用する。フロア管理・オーダー入力・会計・出退勤打刻に特化し、暗い店内でも対応できるダークモードを標準とする。

**対象ユーザー**: ボーイ（フロアスタッフ）

**端末**: iPad（10インチ推奨）

**認証**: 端末認証（店舗端末として登録済みの端末のみアクセス可）。PIN認証は使用しない。

**現状**: React + TypeScript のWebプロトタイプで主要UIとビジネスロジックを検証済み。このプロトタイプを参考設計として Swift + SwiftUI でネイティブに書き直す。

```
│  管理画面（Web）             │  ← 別プロジェクト（luna-admin）
│  Vite + React + TypeScript   │
│  売上分析・設定・給与計算      │

```

### 2.2 POSアプリ（ネイティブ iOS）

```
言語:       Swift
UI:         SwiftUI
ローカルDB: SwiftData（iOS 17+）
通信:       Supabase Swift SDK（supabase-swift）
プリンタ:   Epson ePOS SDK (iOS) / Star SDK
            → Bluetooth・ネットワーク両対応
最小OS:     iOS 17+
```

**ネイティブを選択する理由:**

- **オフライン信頼性**: SwiftDataに即座に書き込み → ネット復旧時にSupabaseへ同期。営業中にネットが落ちてもオーダー入力・会計が止まらない
- **プリンタ連携**: Epson ePOS SDK / Star SDKのネイティブ版でBluetooth・ネットワーク両方に対応。Webでは不可能なBluetooth接続がiPadでも使える
- **操作レスポンス**: 画面遷移・タップ応答がWebより確実に速い。忙しい営業中にわずかな遅延が積もるのを防ぐ

### 2.3 管理画面（Web）— 別プロジェクト

管理画面はオーナー・マネージャーが使用。売上分析・メニュー設定・給与計算・顧客管理など、リアルタイム性やオフライン性が不要な機能をWebで提供する。luna-admin プロジェクトで別途対応。

## 3. 機能要件

### 3.1 テーブル管理（フロア画面）— メイン画面

- **マルチルーム対応**: 複数の部屋（メインルーム・VIPルーム・カウンター等）をスワイプで切り替え
- フロアマップ上にテーブルを配置・表示（8×5グリッド）
- テーブルごとのステータス管理（空席 / 使用中 / 会計待ち）
- ステータスに応じた色分け表示（緑=使用中、琥珀=会計待ち、紫=空席）
- 入店時刻の自動記録 → セット料金の自動計算
- 各テーブルの経過時間をリアルタイム表示
- ヘッダーに本日売上合計を表示

**入店（テーブルオープン）時の入力:**
- 顧客名（任意）
- 人数（+/- で調整）
- 指名キャスト選択（複数可）
- 指名種別（なし / 場内 / 本指名）
- 同伴キャスト選択（任意）
- リアルタイム料金プレビュー表示

### 3.2 オーダー管理

- **2ペインレイアウト**: 左にメニュー一覧、右にオーダー伝票
- カテゴリ別メニュー表示（ドリンク / ボトル / フード / レディースドリンク / その他 / 建て替え）
- カスタムアイテム入力（メニューにない商品の手入力）
- 数量の +/- 調整
- **建て替え（経費）アイテム**: サービス料・消費税の対象外として別計上
- **価格の手動上書き**: セット料金・指名料・同伴料・各オーダーアイテムの価格をタップして変更可能
- **延長管理**: 30分単位の延長追加（人数分の料金自動計算）
- テーブル移動（空きテーブルへの移動）
- 指名の追加・変更（モーダルで編集）
- 人数変更

### 3.3 会計機能

**料金計算ロジック（プロトタイプで検証済み）:**

```
セット料金      = 基本セット + 延長料金（人数 × 延長単価）
指名料合計      = Σ（各キャストの指名料 × 数量）
同伴料          = 同伴料 × 数量
通常オーダー合計 = Σ（各アイテムの単価 × 数量）※建て替え除く

小計           = セット料金 + 指名料合計 + 同伴料 + 通常オーダー合計
サービス料     = floor(小計 × サービス料率)
消費税         = floor(小計 × 税率)
課税後合計     = 小計 + サービス料 + 消費税
最終合計       = 課税後合計 - 割引額 + 建て替え合計
```

**デフォルト料率（店舗設定で変更可能にする）:**

| 項目 | デフォルト値 |
|------|-------------|
| サービス料率 | 40% |
| 消費税率 | 10% |
| 同伴料 | ¥3,000 |
| 本指名料 | ¥5,000 |
| 場内指名料 | ¥2,000 |

**会計画面の機能:**
- 支払い方法の選択（現金 / クレジットカード / 電子マネー / ツケ）
- 割引額入力
- 明細一覧表示 → 修正可能
- レシート発行（Bluetooth or ネットワークプリンタ連携 / PDF出力）
- 会計完了後 → フロア画面に自動遷移

### 3.4 キャスト出退勤打刻

- キャスト一覧をグリッド表示（写真付き）
- ステータス表示:
  - 緑 = 出勤中
  - 金 = シフト予定あり（予定時刻表示）
  - 紫 = 未出勤
- タップで詳細パネル表示:
  - 写真（拡大表示可）
  - 源氏名・本名
  - 出勤 / 退勤ボタン
  - シフト予定
  - 送り先
  - 本日の指名数・売上
- キャスト新規追加（写真アップロード付き）
- 打刻はPOS端末からのみ可能（キャスト本人は打刻できない）
- 出勤打刻 → ホームページに自動反映（Realtime連携）

### 3.5 レジ精算・日次レポート

**プロトタイプで実装済みの機能:**
- 本日の売上合計・来店組数・客単価・平均滞在時間
- 支払い方法別の内訳
- **レジ精算**: スタート金額設定 → 現金売上 → 出金記録 → 理論残高の算出
- 出金記録の追加（金額・メモ）
- キャスト別売上ランキング
- 会計履歴一覧

### 3.6 設定・マスタ管理（POS内簡易版）

**プロトタイプで実装済みの機能:**
- **ルーム管理**: ルームの追加・編集・削除
- **テーブル管理**: テーブルのルーム間移動、グリッド上の位置調整
- **メニュー管理**: メニュー追加、カテゴリ設定、有効/無効切替
- **キャスト管理**: キャスト追加・編集、写真登録

> 注意: これらの設定機能は暫定的にPOS内に含めている。将来的には管理画面（luna-admin）に移行し、POSは参照のみとする。

### 3.7 顧客簡易参照

- テーブルに顧客を紐付ける際の検索・選択
- 過去の来店回数・ボトルキープ状況の簡易表示
- 詳細な顧客管理は管理画面で行う

- **ダークモード標準**（暗い店内での使用を前提）

- 高コントラスト配色（紫/金のカラースキーム）

- iPad横向き固定を推奨レイアウトとする

## 8. オフライン設計

営業中にネットが落ちても業務を止めないことが最優先。

### 8.1 基本方針

```
SwiftDataを正とし、常にローカルに先行書き込み
→ ネット接続時にSupabaseへバックグラウンド同期
→ 競合発生時はタイムスタンプベースで解決（最終更新優先）
```

### 8.2 オフライン時に可能な操作

- テーブルのステータス変更（空席 → 使用中 → 会計待ち）
- オーダー入力・追加・数量変更
- 会計計算・支払い記録
- キャスト出退勤打刻
- 指名の紐付け・変更
- 価格の手動上書き
- 延長追加
- レジ精算操作

### 8.3 オフライン時に制限される操作

- 他端末とのリアルタイム同期（復旧後に反映）
- HP出勤情報の自動反映（復旧後に反映）
- 顧客検索（ローカルにキャッシュ済みのデータのみ参照可）

### 8.4 同期戦略

- 各レコードに `synced` フラグと `updatedAt` タイムスタンプを保持
- NWPathMonitor でネット復旧を検知 → `synced = false` のレコードを一括アップロード
- 複数端末からの同時書き込みは `updatedAt` の新しい方を採用
- 読み取り専用テーブルはアプリ起動時にローカルDBへキャッシュし、Supabase Realtimeで差分更新

## 12. 注意事項

- 営業中の操作画面なので**スピード最優先**。分析・設定の高度な機能は管理画面で行う
- メニューや料金の変更は最終的には管理画面（Web）で行い、POSは参照のみとする（Phase 2で移行）
- 出退勤の「打刻」はPOSで行うが、「修正」は管理画面の権限
- プリンタの接続テスト（Bluetooth / ネットワーク）は実機で行うこと
- iOS先行開発。Android対応は将来検討（Kotlin + Jetpack Compose）
- オフライン同期の競合解決ロジックは、実運用テストで十分に検証すること
- SwiftDataは iOS 17+ が必要 — 対応OSバージョンに注意
- プロトタイプのビジネスロジック（AppContext.tsx内の28アクションのリデューサー）は動作検証済みなので、Swiftに忠実に移植すること
